# IoT Data Platform

### Цель проекта

Цель нашего проекта — создать универсальную платформу для IoT-систем обработки данных.
Многие IoT-проекты имеют схожую структуру: сбор данных, потоковая обработка, генерация аналитических отчётов в пакетном режиме, хранение данных и предоставление информации пользователям.
Команды, реализующие такие проекты, смогут использовать нашу платформу вместо создания инфраструктуры с нуля.

### Проблема

Сегодня существует множество IoT-проектов: например, отслеживание самокатов и велосипедов, мониторинг качества воздуха и т.д.
Небольшой команде сложно собрать экспертов во всех областях: системы сбора данных, потоковая и пакетная обработка, хранение, предоставление данных, мониторинг и развёртывание.

Изучение всех инструментов и их интеграция занимают очень много времени.

Мы планируем объединить готовые open source-системы в одну платформу:
- **Kafka** - приём входящих данных (ingest)
- **Flink** - потоковая обработка
- **dbt** - пакетная обработка (batch)
- **Redis** - быстрые низколатентные чтения
- **PostgreSQL** - реляционные данные
- **S3-совместимое хранилище** - исторические данные
- **Elasticsearch / Logstash / Kibana** - мониторинг и логирование
- **Kubernetes / Docker Compose** - развёртывание

С готовой платформой команды смогут сосредоточиться на доменных задачах и бизнес-логике, а не на борьбе с инструментами и интеграцией.

## Архитектура данных

<img width="1432" height="663" alt="image" src="https://github.com/user-attachments/assets/990342f2-a57c-454c-9cb7-707dee221854" />

В основе лежит хранилище данных - сюда попадают как “сырые” события, так и результаты обработки.

Слева - блок Ingest: он отвечает за приём данных из внешних источников, будь то датчики, транспорт или другие сервисы.

В центре - Transform: здесь данные очищаются, обогащаются и агрегируются как в потоковом, так и в пакетном режиме.

Справа - Serve: этот слой отвечает за доступ к данным - для API, дашбордов, аналитики и любых пользовательских приложений.

В рамках разработки платформы предусмотрены и реализуются несколько сценариев, демонстрирующих один и тот же каркас (ingest/transform/serve), но с разными доменами данных:
- **Автобусы**: обработка прибытий:
  - https://github.com/IoTDataPlatform/IoT_Data_Platform_transport_scenario
  - https://github.com/IoTDataPlatform/transport_demo_project_backend
  - https://github.com/IoTDataPlatform/transport_demo_project_frontend
- **Датчики**: оконные агрегации и подсчёт “сухих дней”
  - https://github.com/IoTDataPlatform/pvz_demo_project
- **RICH detector**: фильтрация колец:
  - https://github.com/IoTDataPlatform/richgen_demo_project
